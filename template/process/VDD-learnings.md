# VDD 運用ログ（学びの蓄積）

VDD を運用する中で得た学び・気づき・改善点を蓄積する場所。
VDD.md（仕様）を更新する際の参照材料として使う。

## 記録ルール

- 日付と文脈を必ず記録する
- 「何が起きたか」「なぜそうなったか」「次にどうすべきか」の3点を含める
- VDD.md への反映が必要な場合は、反映後にその旨を追記する

## 推奨フォーマット

```markdown
## YYYY-MM-DD: タイトル

**文脈**: どの作業で発見したか
**何が起きたか**: 事実
**なぜそうなったか**: 原因分析
**次にどうすべきか**: 対策・提案
```
